# 答辩演示流程（模块12）

## 1. 演示目标
本流程用于毕业答辩现场演示，覆盖：
1. 系统可运行性
2. 核心业务闭环（认证、文件、权限、审计）
3. 安全能力（SQL注入、XSS、越权）
4. 测试与性能结果

建议总时长：15-20 分钟。

## 2. 演示前准备（5 分钟）

### 2.1 环境确认
1. 启动 MySQL 服务
2. 激活 Python 虚拟环境
3. 在项目根目录执行：
```bash
python init_db.py
python insert_admin.py
python upgrade_module8.py
python upgrade_module9.py
python upgrade_module11.py
```

### 2.2 启动系统
```bash
python app.py
```

打开浏览器：
1. `http://127.0.0.1:5000/`
2. `http://127.0.0.1:5000/login`

## 3. 演示主流程

### 3.1 系统启动与基础页面（约 2 分钟）
1. 展示首页与登录入口
2. 说明整体架构：Flask + SQLAlchemy + MySQL + 生物识别 + ABAC + 审计
3. 进入登录页，强调管理员与用户两类入口

### 3.2 管理员侧演示（约 4 分钟）
1. 使用管理员账号登录
2. 进入“系统管理”页面，创建一个普通用户
3. 在“权限管理”中调整该用户安全级别
4. 在“文件密级设置”中修改某文件密级
5. 进入“审计日志”页面，按操作类型过滤并导出 CSV

讲解重点：
1. 权限变更全量审计
2. 管理员专属路由受保护

### 3.3 普通用户侧演示（约 4 分钟）
1. 使用普通用户执行口令 + 人脸 + 指纹三阶段认证
2. 登录后进入文件页上传文件（自动加密）
3. 下载文件（自动解密）
4. 保存新版本并查看历史版本
5. 尝试访问高密级文件，展示“权限不足”

讲解重点：
1. 双因子串行认证
2. ABAC 控制可见范围与下载权限
3. 文件独立密钥机制

### 3.4 安全测试演示（约 3 分钟）
1. SQL注入：在登录页输入注入语句，演示无法绕过认证
2. XSS：构造可疑输入（文件名/查询参数），页面不执行脚本
3. 越权：普通用户访问 `/audit/logs` 或 `/abac/permissions` 被拦截

可直接运行脚本：
```bash
python -X faulthandler tests/module12_security_check.py
```

### 3.5 全局测试与性能演示（约 3 分钟）
1. 运行全量回归：
```bash
python -X faulthandler tests/module12_full_check.py
```
2. 说明输出中包含：
   - 模块5/8/9/10/11回归结果
   - 模块12整合/安全/性能结果
3. 展示性能结果（页面平均响应、上传下载吞吐）

## 4. 答辩讲解提纲（可照读）
1. 本系统完成了“认证-授权-存储-审计”全链路闭环。
2. 在访问控制上，使用 ABAC 将用户安全级别与文件密级绑定。
3. 在数据安全上，文件采用 AES-256-CBC 并为每条记录分配独立密钥。
4. 在审计安全上，日志具备不可改删约束并支持完整追踪导出。
5. 在模块12阶段，通过整合测试、安全测试、性能测试确认系统可交付。

## 5. 备用演示方案（网络或摄像头异常时）
1. 跳过实时摄像头采集，使用预存图片走验证流程
2. 若现场网络限制，使用本机回环地址与本地数据库演示
3. 优先演示测试脚本输出，证明核心能力可重复验证
